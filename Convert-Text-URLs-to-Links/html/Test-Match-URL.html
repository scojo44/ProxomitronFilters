This is a page for testing filters that convert text-only URLs to clickable links. The slow version of the Match_URL list should match the ones the fast one doesn't.

Real world test pages:
http://www.virtualsalt.com/warning.htm - Several full URLs
http://grc.com/dos/attacklog.htm - Lots of IP addresses and hostnames 

E-mail address tests
--------------------
i@i.il
billg@microsoft.com
billg@microsoft.com. - Period should not be in the link
mailto:billg@microsoft.com
billg@microsoft.com   - billg&#064;microsoft.com
billg@microsoft.com  - bill<span style="font-weight:bold;">g&#064;micro</span>soft.com

Hostname tests
--------------
www.mail.netzero.com       - should match the whole hostname
foo.net.com                - should match the whole hostname
www....foo.com             - shouldn't match?
roadsign.jpg               - shouldn't match (filename)
comp.os.linux.networking   - shouldn't match (newsgroup)
.com .net .co.uk           - first two shouldn't match.

IP address tests
----------------

@10.141.233.173

----------------

Protect list tests
------------------
URLs in // comments shouldn't match.
www.wrong.com       // Already existing link should go to www.right.com

x www.wrong.com
then, http://www.wrong.net/
// the above two lines should link to www.right.com

CSS: The url() part shouldn't match
body {background-image: url(http://www.example.com/back.gif)}

JavaScript: object.name and www.example.com shouldn't
match in the below if() line.
if(checked && (object.name == 'www.example.com'))
